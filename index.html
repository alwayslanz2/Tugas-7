Menavigasi Antarmuka Winbox Dan Cli Pada Mikrotik

a. Login Winbox

Gambar 1: Tampilan awal Neighbor

Sumber foto: m adlan ali

Menurut biznethome.net, proses login di Winbox dapat dilakukan dengan membuka aplikasi, lalu masuk ke tab Neighbors. Tab ini berfungsi mendeteksi perangkat MikroTik yang terhubung melalui jaringan. Dari daftar perangkat, pengguna dapat memilih perangkat berdasarkan MAC Address, kemudian mengisi username (default: admin) serta password (kosong pada kondisi awal), lalu menekan tombol Connect untuk masuk ke sistem.

Menurut IDN-Networkers, login juga bisa dilakukan melalui fitur Connect To. Pada bagian ini, pengguna bisa memasukkan langsung alamat IP atau MAC Address perangkat. Selain itu, terdapat tombol Add/Set yang memungkinkan pengguna menyimpan informasi login sehingga tidak perlu mengetik ulang di kemudian hari.

Menurut tutorialmikrotik.com, Winbox sendiri adalah aplikasi GUI (Graphical User Interface) yang dirancang agar konfigurasi MikroTik lebih mudah digunakan, terutama oleh pemula yang tidak terbiasa dengan perintah berbasis teks. Fitur tambahan seperti Neighbor Discovery untuk mendeteksi perangkat, Safe Mode untuk mencegah kesalahan fatal, serta Add/Set Managed untuk menyimpan data login membuat proses login semakin efisien.

Dari beberapa pendapat tersebut dapat disimpulkan bahwa login Winbox sangat user-friendly karena mendukung pemindaian otomatis perangkat (Neighbor Discovery), pilihan login via IP atau MAC, serta penyimpanan data login untuk memudahkan akses berikutnya./p>

b. Login CLI

Gambar 2: Tampilan awal CLI

Sumber foto: M Adlan Ali

Menurut Herza.ID, bagi pengguna MikroTik Cloud Hosted Router (CHR) di VPS, login CLI (Command Line Interface) bisa dilakukan melalui berbagai cara. Beberapa metode yang umum dipakai yaitu menggunakan VNC Panel (Web Console), SSH, maupun terminal di Winbox. Pilihan metode ini membantu ketika Winbox tidak bisa diakses secara langsung karena adanya pemblokiran port atau keterbatasan jaringan.

Menurut Citraweb, CLI memiliki fitur khusus bernama Menu Setup yang bisa dijalankan langsung melalui perintah setup di terminal. Dari menu ini, pengguna dapat melakukan konfigurasi dasar seperti pengaturan DHCP, IP Address, PPPoE, hingga reset konfigurasi. Menariknya, saat masuk ke mode setup, CLI otomatis berjalan dalam kondisi Safe Mode sehingga konfigurasi bisa lebih aman.

Menurut Labkom.co.id, CLI menyediakan berbagai perintah dasar yang penting untuk administrasi jaringan. Contohnya perintah ip address add untuk menambahkan alamat IP, ip address enable/disable untuk mengaktifkan atau menonaktifkan interface, serta export untuk membuat backup konfigurasi agar mudah dipulihkan jika terjadi masalah.

Dari beberapa pendapat tersebut dapat disimpulkan bahwa login dan penggunaan CLI memberikan fleksibilitas tinggi dalam mengakses MikroTik, karena bisa dilakukan melalui berbagai metode (VNC, SSH, terminal) sekaligus mendukung konfigurasi dasar maupun lanjutan langsung dari antarmuka teks.

c. Menu Utama Winbox

Gambar 3: tampilan menu utama winbox

Sumber foto: m adlan ali

Menurut biznethome.net, setelah pengguna berhasil login, tampilan utama Winbox langsung menampilkan menu penting. Beberapa di antaranya adalah Interfaces (untuk melihat daftar port fisik dan virtual), IP (berisi pengaturan IP Address, DHCP, Firewall, NAT), serta Bridge (untuk menghubungkan beberapa interface dalam satu jaringan).

Menurut IDN-Networkers, selain menyediakan opsi Connect To pada awal login, Winbox juga menghadirkan menu navigasi utama seperti Interfaces, Bridge, dan IP. Menu-menu ini langsung terlihat begitu masuk, sehingga memudahkan administrator melakukan konfigurasi awal jaringan.

Menurut tutorialmikrotik.com, Winbox juga dilengkapi dengan fitur tambahan yang muncul di menu utama, seperti Neighbor Discovery untuk mendeteksi perangkat jaringan sekitar, Safe Mode untuk menjaga konfigurasi tetap aman saat melakukan perubahan, serta Add/Set Managed untuk mengelola data login perangkat yang sudah tersimpan.

Dari beberapa pendapat tersebut dapat disimpulkan bahwa menu utama Winbox dirancang intuitif dan kaya fitur, memungkinkan pengguna langsung mengakses fungsi penting seperti Interfaces, IP, Bridge, serta fitur manajemen tambahan seperti Safe Mode dan Managed login.

d. Dasar Perintah CLI

Gambar 4: langkah awal CLI

Sumber foto: M Adlan Ali

Menurut Labkom.co.id, perintah dasar pada CLI MikroTik yang sering digunakan adalah ip address add untuk menambahkan alamat IP, ip address print untuk menampilkan daftar IP yang sudah terpasang, ping untuk melakukan pengecekan konektivitas, interface print untuk melihat daftar interface yang tersedia, serta export yang berguna untuk membuat backup konfigurasi dalam bentuk teks agar bisa disimpan atau digunakan kembali.

Menurut Citraweb, penguasaan perintah dasar di CLI membantu administrator dalam melakukan konfigurasi awal, seperti menyiapkan DHCP server, menetapkan IP Address pada interface, mengonfigurasi PPPoE, hingga melakukan reset konfigurasi router ketika diperlukan. Semua proses tersebut bisa dilakukan dengan cepat jika pengguna terbiasa dengan sintaks CLI.

Menurut Herza.ID, pemahaman tentang sintaks perintah seperti /ip address, /interface, dan /ping adalah hal mendasar yang wajib dikuasai sebelum melangkah ke konfigurasi lanjutan. Hal ini karena CLI memberikan fleksibilitas penuh dalam mengontrol RouterOS dan menjadi fondasi untuk fitur-fitur jaringan yang lebih kompleks.

Dari beberapa pendapat tersebut dapat disimpulkan bahwa perintah dasar CLI MikroTik merupakan fondasi administrasi router, mencakup konfigurasi IP, pengelolaan interface, serta pengecekan koneksi dengan ping sebelum masuk ke pengaturan yang lebih rumit.

e. Navigasi Interface

Gambar 5: menampilkan interface lewat CLI

Sumber foto: M Adlan Ali S

Menurut Biznethome.net, menu Interfaces di Winbox menampilkan daftar semua port fisik seperti ether1, ether2, dan seterusnya, serta interface virtual seperti bridge, VLAN, atau tunnel. Dari menu ini, administrator dapat langsung melihat struktur jaringan yang terhubung ke perangkat.

Menurut IDN-Networkers, navigasi pada tab Interfaces di Winbox sangat membantu untuk memantau status port apakah dalam kondisi running atau tidak. Selain itu, fitur traffic realtime memungkinkan pengguna mengamati data yang lewat pada setiap port sehingga memudahkan analisis trafik.

Menurut Labkom.co.id, pada CLI tersedia perintah /interface print untuk menampilkan daftar interface yang ada, serta /interface set untuk melakukan konfigurasi manual, misalnya mengaktifkan atau menonaktifkan port tertentu. Hal ini memudahkan pengelolaan jaringan tanpa harus menggunakan antarmuka grafis.

Dari beberapa pendapat tersebut dapat disimpulkan bahwa navigasi interface adalah kunci untuk mengelola port jaringan, baik dengan memanfaatkan menu Winbox yang lebih visual maupun perintah CLI yang lebih teknis.

f. Konfigurasi IP Address

Sumber foto: m adlan ali

Menurut Citraweb, konfigurasi IP Address termasuk bagian dari Basic Setup yang wajib dilakukan di tahap awal. Tanpa adanya IP Address, perangkat MikroTik tidak bisa berkomunikasi dengan perangkat lain di jaringan, sehingga layanan lain seperti DHCP, NAT, atau firewall tidak akan berjalan.

Menurut Biznethome.net, konfigurasi IP juga dapat dilakukan dengan mudah melalui Winbox, tepatnya pada menu IP > Addresses. Administrator hanya perlu menambahkan IP Address pada interface tertentu tanpa harus mengetikkan perintah secara manual, sehingga lebih ramah bagi pengguna pemula.

Dari beberapa pendapat tersebut dapat disimpulkan bahwa konfigurasi IP Address merupakan langkah awal paling penting pada MikroTik, karena menjadi dasar agar router dapat terhubung dan berkomunikasi dengan jaringan.

g. Pengaturan DHCP

Menurut Biznethome.net, pengaturan DHCP di MikroTik melalui Winbox dilakukan dengan membuka menu IP > DHCP Server > Setup, kemudian memilih interface yang akan menjadi jalur distribusi alamat IP. Setelah itu, sistem secara otomatis akan memandu admin dalam menentukan gateway, DNS server, serta range IP yang akan dibagikan kepada klien.

Menurut IDN-Networkers, DHCP dapat diaktifkan dengan cara memilih interface LAN yang terhubung ke jaringan lokal. Admin kemudian menentukan rentang alamat IP sehingga perangkat klien yang terkoneksi dapat langsung memperoleh alamat IP tanpa harus diatur manual satu per satu.

Menurut Herza.ID, pada CLI konfigurasi DHCP dilakukan dengan perintah /ip dhcp-server setup. Setelah dijalankan, MikroTik akan memberikan sejumlah pertanyaan kepada admin, seperti network yang digunakan, alamat gateway, range IP, dan DNS server, sehingga seluruh konfigurasi dapat dilakukan secara otomatis tanpa perlu proses panjang.

Dari beberapa pendapat tersebut dapat disimpulkan bahwa pengaturan DHCP di MikroTik berfungsi mempermudah distribusi alamat IP secara otomatis kepada perangkat klien, baik melalui Winbox dengan antarmuka grafis maupun melalui CLI dengan perintah langsung.

h. Monitoring Trafik

Menurut IDN-Networkers, fitur Torch pada Winbox digunakan untuk memantau trafik jaringan secara real-time. Dengan fitur ini, admin bisa melihat IP sumber, IP tujuan, port, serta protokol yang sedang berjalan di jaringan, sehingga mempermudah analisis penggunaan bandwidth.

Menurut Biznethome.net, monitoring trafik juga dapat dilakukan melalui menu Interfaces pada Winbox. Pada bagian ini, admin dapat memperhatikan nilai RX (receive) dan TX (transmit) di setiap port, yang menunjukkan seberapa besar bandwidth sedang dipakai oleh masing-masing interface.

Menurut Labkom.co.id, pada CLI, admin dapat menggunakan perintah /tool torch atau /interface monitor-traffic untuk melihat aktivitas data, bandwidth yang sedang digunakan, dan performa interface tertentu. Hal ini memudahkan admin dalam menganalisis lalu lintas jaringan tanpa harus menggunakan aplikasi tambahan.

Dari beberapa pendapat tersebut dapat disimpulkan bahwa monitoring trafik sangat penting untuk mengetahui pemakaian bandwidth dan aktivitas jaringan secara real-time, baik dengan antarmuka GUI (Winbox) maupun perintah CLI.

i. Troubleshooting Dasar

Sumber foto: m adlan ali

Menurut Citraweb, perintah ping dan traceroute pada MikroTik digunakan untuk menguji konektivitas antar perangkat serta menelusuri rute komunikasi jaringan. Dengan perintah ini, administrator dapat memastikan apakah jalur komunikasi jaringan berfungsi dengan baik atau terdapat hambatan pada titik tertentu.

Menurut Labkom.co.id, perintah log print di CLI sangat penting karena berfungsi untuk menampilkan catatan aktivitas sistem, termasuk error. Dengan membaca log ini, administrator dapat mengetahui letak gangguan, sehingga perbaikan dapat dilakukan dengan cepat dan tepat sasaran.

Menurut IDN-Networkers, pada aplikasi Winbox tersedia menu Log yang menyajikan status aktivitas router secara langsung. Hal ini membantu administrator dalam memantau sistem tanpa harus menggunakan perintah CLI, sehingga lebih praktis bagi pengguna yang terbiasa dengan antarmuka grafis.

Dari beberapa pendapat tersebut dapat disimpulkan bahwa troubleshooting dasar pada MikroTik mencakup pengujian koneksi (ping, traceroute) untuk memastikan jalur jaringan, serta pemeriksaan log melalui CLI maupun Winbox untuk mendeteksi dan menganalisis sumber gangguan.

j. Langkah Awal Konfigurasi MikroTik setelah Login Pertama

Sumber foto: m adlan ali

Menurut Modalsemangat, langkah awal konfigurasi MikroTik berfokus pada pengaturan jaringan internal. Administrator biasanya menetapkan IP Address dan subnet mask pada interface LAN, sehingga router dapat berfungsi sebagai penghubung utama perangkat di jaringan lokal. Selain itu, administrator sering menambahkan DHCP Server, agar setiap perangkat yang terhubung secara otomatis mendapat alamat IP tanpa harus diatur secara manual, sehingga mempermudah manajemen jaringan.

Menurut Cyberlink, selain jaringan lokal, administrator juga perlu melakukan konfigurasi pada sisi WAN agar router dapat terhubung ke internet. Hal ini bisa menggunakan DHCP Client jika ISP memberikan IP secara otomatis, atau PPPoE jika penyedia layanan internet mensyaratkan autentikasi. Setelah koneksi internet aktif, administrator perlu mengatur DNS Server supaya proses resolusi domain berjalan lancar. Langkah berikutnya adalah menambahkan NAT (Network Address Translation) agar semua perangkat di jaringan lokal bisa berbagi koneksi internet menggunakan satu alamat IP publik. Untuk melindungi jaringan, administrator juga dianjurkan menambahkan firewall dasar, misalnya untuk memblokir akses dari luar yang tidak diperlukan.

Menurut Scribd, selain konfigurasi teknis, administrator pemula disarankan segera melakukan pengamanan router. Hal ini mencakup mengganti username dan password default dengan kombinasi yang lebih kuat, serta menonaktifkan service yang tidak penting seperti Telnet, FTP, atau API yang rawan dimanfaatkan oleh pihak tidak bertanggung jawab. Langkah pengamanan ini sangat penting karena konfigurasi awal biasanya dilakukan langsung setelah perangkat baru diaktifkan, sehingga potensi serangan dari luar bisa diminimalisir sejak awal.

Dari beberapa pendapat tersebut dapat disimpulkan bahwa langkah awal konfigurasi MikroTik setelah login pertama mencakup Pengaturan jaringan lokal melalui IP Address, subnet mask, dan DHCP server. Konfigurasi koneksi internet (WAN) menggunakan DHCP Client atau PPPoE. Pengaturan DNS, NAT, dan firewall dasar untuk memastikan jaringan berjalan aman dan stabil. Pengamanan router dengan password kuat, mengganti login default, serta menonaktifkan service yang tidak diperlukan.

k. Perbandingan Winbox vs CLI

Sumber foto: m adlan ali

Menurut Biznethome.net, Winbox lebih ramah digunakan oleh pemula karena berbasis GUI (Graphical User Interface). Sedangkan CLI (Command Line Interface) lebih cepat dioperasikan oleh administrator yang sudah terbiasa dengan perintah teks, terutama untuk konfigurasi jaringan yang berulang.

Menurut Herza.ID, CLI dinilai lebih unggul karena memberikan fleksibilitas penuh kepada administrator. CLI mendukung scripting dan konfigurasi kompleks, sehingga memudahkan otomatisasi dalam pengelolaan jaringan berskala besar.

Menurut Citraweb, kelebihan utama Winbox terletak pada sisi visualisasi konfigurasi, sehingga memudahkan monitoring jaringan secara grafis. Sementara itu, CLI lebih ringan, hemat resource, dan sangat cocok digunakan untuk remote management melalui terminal.

Dari beberapa pendapat tersebut dapat disimpulkan bahwa Winbox lebih cocok untuk pemula dan memudahkan dalam visualisasi konfigurasi, sedangkan CLI lebih fleksibel, efisien, serta sesuai bagi admin berpengalaman yang membutuhkan kontrol penuh terhadap sistem.
